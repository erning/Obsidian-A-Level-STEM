# Experimental Design and Data Analysis Worked Examples

These examples model how to think through [[10 Physics/02 Experimental Thinking/Experimental Design and Data Analysis/00 Overview|Experimental Design and Data Analysis]] problems. They are solution patterns, not a substitute for the practice set.

## Example 1: Method for Experimental Design and Data Analysis

**Prompt.** Work with this syllabus idea: Define independent, dependent, and control variables.

**Solution.**

1. Name the relevant concept from Experimental Design and Data Analysis and write any definitions that control it.
2. Identify the quantities, variables, graph features, events, or geometric objects involved.
3. Choose the representation that makes the relation simplest.
4. Carry out the algebra, calculation, or interpretation without changing the model halfway through.
5. Check the result. For physics, keep three checks visible: units, direction or sign, and the modelling assumptions behind the equation.

**What this example trains.** Method choice, clean setup, and checking the answer against the structure of the topic.

## Example 4: Recovering from a Common Trap

**Prompt.** A solution to a Experimental Design and Data Analysis problem goes wrong because of this trap: Listing generic improvements that do not address the actual limitation.

**Solution.**

1. Locate the step where the condition, sign, unit, domain, or modelling assumption was lost.
2. Return to the definition and rewrite the setup with that condition included.
3. Redo only the affected part of the calculation.
4. Compare the corrected result with the original one and state why the corrected version is valid.

This repair habit is often more valuable than memorising another formula.
