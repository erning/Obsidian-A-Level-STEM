# Experimental Design and Data Analysis Review Checklist

Use this checklist after the practice problems. A Paper 5 answer must make the
experiment workable, the analysis linear, and the conclusion tied to
uncertainty.

## Ideas to Recall

- [ ] The independent variable is deliberately changed.
- [ ] The dependent variable is measured as the outcome.
- [ ] Control variables must be named and controlled by a stated method.
- [ ] A workable plan must allow the required data to be collected without
      undue difficulty.
- [ ] A clear labelled diagram supports the written method.
- [ ] A wide range and enough readings improve the graph and reveal scatter.
- [ ] Linearisation turns a model into a graph with meaningful gradient and
      intercept.
- [ ] For $y=ax^n$, plot $\lg y$ against $\lg x$.
- [ ] For $y=ae^{kx}$, plot $\ln y$ against $x$.
- [ ] Error bars show uncertainty in plotted points.
- [ ] A worst acceptable line is the steepest or shallowest line through the
      error bars.
- [ ] A conclusion compares the result with the model or accepted value using
      uncertainty.
- [ ] A limitation must name the measurement or control that weakens the
      result.
- [ ] An improvement must address a specific limitation.
- [ ] A safety point must name a hazard and a matching precaution.

## Skills to Perform

- [ ] Identify independent, dependent, and control variables.
- [ ] State how each control variable is kept constant.
- [ ] Describe how the independent variable is varied and measured.
- [ ] Describe how the dependent variable is measured and repeated.
- [ ] Choose apparatus and sensors that are fit for purpose.
- [ ] Rearrange a relationship into $y=mx+c$ form.
- [ ] Decide which derived quantities to calculate for the graph.
- [ ] Match gradient and intercept to physical constants.
- [ ] Use best-fit and worst acceptable lines to estimate uncertainty.
- [ ] Quote constants with value, uncertainty, and unit.
- [ ] Compare percentage difference with percentage uncertainty.
- [ ] Write specific limitation-improvement pairs.
- [ ] Describe use of oscilloscopes, light gates, and data loggers where
      appropriate.

## Common Errors to Avoid

- [ ] Naming a control variable without saying how it is held constant.
- [ ] Using a narrow range of independent-variable values.
- [ ] Choosing apparatus that cannot measure the required quantity precisely
      enough.
- [ ] Linearising with the wrong logarithm or plotting the wrong axes.
- [ ] Writing $\lg d$ instead of $\lg(d/\mathrm{cm})$ in a table heading.
- [ ] Drawing a worst acceptable line that misses one or more error bars.
- [ ] Reading gradients from points that are too close together.
- [ ] Quoting a final constant without uncertainty or unit.
- [ ] Claiming the data prove a relationship.
- [ ] Writing generic limitations such as "human error" without detail.
- [ ] Suggesting an improvement that does not address the limitation.
- [ ] Writing "be careful" instead of a real safety precaution.

## Ready to Move On When

- [ ] I can design a workable experiment from a given relationship.
- [ ] I can state variables, controls, apparatus, method, range, repeats, graph,
      and safety.
- [ ] I can linearise linear, power-law, and exponential relationships.
- [ ] I can use error bars and worst acceptable lines to estimate uncertainty.
- [ ] I can draw conclusions from uncertainty rather than from expectation.
- [ ] I can write useful evaluations with realistic improvements.
