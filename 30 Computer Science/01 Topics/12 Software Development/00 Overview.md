# 12 Software Development

## Core Idea

Code that works has to be designed, tested, and maintained. This topic covers the development life cycle, the diagrams used to design programs, and the methods used to find and fix faults.

## Source Alignment

- 9618 AS section 12 Software Development
- 12.1 Program Development Life cycle

- 12.3 Program Testing and Maintenance
- Paper 2

## What to Learn

- Purpose of a development life cycle and the need for different cycles (waterfall, iterative, rapid application development (RAD)); their principles, benefits, and drawbacks.
- Analysis, design, coding, testing, and maintenance stages.
- Structure charts: decomposing a problem into sub-tasks and showing parameters passed between modules.
- Deriving pseudocode from a structure chart.
- State-transition diagrams to document an algorithm.
- Error types: syntax, logic, run-time.
- Testing methods: dry run, walkthrough, white-box, black-box, integration, alpha, beta, acceptance, stub.
- Test strategy and test plan; choosing normal, abnormal, and extreme/boundary data.
- Maintenance types: perfective, adaptive, corrective.
- Analysing an existing program and amending it to enhance functionality.

## How to Study

- Match each life cycle type to a kind of project (well-understood versus changing requirements).
- Read a structure chart top-down and trace the parameter flow.
- Select test data before running, covering normal, abnormal, and boundary cases.

## Practice Directions

- Construct a structure chart for a given problem.
- Choose a testing method and suitable test data for a given routine.
- Classify a maintenance task as perfective, adaptive, or corrective.

## Common Traps

- Choosing only normal test data and missing boundary cases.
- Mixing up alpha and beta testing.
- Classifying corrective maintenance as adaptive.
