# 09 Algorithm Design and Problem-solving

## Core Idea

Computational thinking turns a problem into something a computer can do. This topic covers abstraction, decomposition, and the basic notation (structured English, flowcharts, pseudocode) used to express solutions.

## Source Alignment

- 9618 AS section 9 Algorithm Design and Problem-solving
- 9.1 Computational Thinking Skills
- 9.2 Algorithms
- Paper 2 (answers in pseudocode per the Pseudocode Guide)

## What to Learn

- Abstraction: need, purpose, benefits, and producing an abstract model that keeps only essential details.
- Decomposition into sub-problems leading to modules (procedures and functions).
- Identifier tables to represent the data a problem uses.
- Pseudocode containing input, process, and output.
- The three constructs: sequence, selection, and iteration.
- Documenting an algorithm with structured English, a flowchart, or pseudocode.
- Converting between structured English, flowcharts, and pseudocode.
- Stepwise refinement to a level of detail that can be programmed.
- Logic statements to define parts of an algorithm.

## How to Study

- Start every problem by identifying the inputs, the process, and the outputs.
- Express the same algorithm in all three notations before settling on pseudocode.
- Refine top-down: write the high-level steps, then expand each one.

## Practice Directions

- Produce an abstract model of a described system.
- Convert a flowchart into pseudocode and vice versa.
- Use stepwise refinement to reach implementable pseudocode.

## Common Traps

- Writing solutions without identifying inputs and outputs.
- Skipping a construct (using iteration where selection is needed).
- Using pseudocode that does not match the official guide.
