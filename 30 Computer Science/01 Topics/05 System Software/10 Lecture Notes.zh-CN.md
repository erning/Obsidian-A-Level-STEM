# System Software 中文讲义

硬件本身是死的：它能开关、能存储，但它不会把资源分配给多个程序、不会管理文件，也不会把人写的文本翻译成它能执行的指令。系统软件就是填补这一空白的那一层。它有两大部分：操作系统，负责替机器上所有程序管理硬件、提供运行环境；语言翻译器，负责把源代码转换成处理器能执行的形式。这一节讲清楚它们各自做什么，以及为什么区分几种翻译器在实际开发中很重要。

## 1. 为什么计算机需要操作系统

操作系统（Operating System，OS）是一种系统程序，它管理硬件，并为应用程序提供运行环境。没有它，每个应用程序都得自己写读取键盘、写磁盘、申请内存、驱动屏幕的代码，而且没有任何两个程序能安全地共用同一台机器。OS 把硬件藏在一个稳定接口后面，应用程序面向这个接口编程，而不是面向裸机；它还公平、安全地在运行的程序之间分配处理器、内存和设备。

### 五项管理任务

OS 执行五项管理任务。要把它们分开：每项管理的是不同的资源。

| 任务 | 管理对象 | 做什么 |
|---|---|---|
| 内存管理（memory management） | RAM 和（通过虚拟内存）外存 | 把 RAM 分配给程序，使其有空间加载和运行；程序结束后回收；防止一个程序覆盖另一个程序的内存 |
| 文件管理（file management） | 外存上的数据 | 让数据以命名文件的形式保存、组织成目录；维护文件名、属性和文件分配表，使文件能被查找、打开和保护 |
| 安全管理（security management） | 访问与完整性 | 控制谁能登录、每个用户能读、写、运行什么；提供密码和访问权限；防范未授权访问和数据丢失 |
| 硬件管理（hardware management，输入输出 / 外设） | 键盘、屏幕、打印机、磁盘、网络 | 提供设备驱动程序，使 OS 和应用程序能向外设收发数据，而不必让每个程序都了解设备的细节 |
| 进程管理（process management） | 正在运行的程序（进程） | 决定哪个进程何时使用处理器（调度），使多个程序看起来在同时运行，每个都能公平地分到 CPU 时间 |

共同的主线是资源共享与保护。OS 是唯一的权威，决定谁拿到内存、谁拿到处理器、谁可以碰哪个文件和设备，并让程序彼此隔离，使其中一个出错不会拖垮其他程序。

## 2. 实用程序

实用程序（utility software）是执行某项具体、通常是日常维护任务的软件，一般与计算机系统的管理和维护有关。大多数实用程序是随 OS 一起提供、而非内核的一部分，每个只做一件事。

| 实用程序 | 做什么 |
|---|---|
| 磁盘格式化（disk formatter） | 把新磁盘（或分区）准备好供使用：划分为磁道和扇区，并在上面建立文件系统，使文件能被存储和定位 |
| 病毒检查（virus checker） | 扫描文件和内存中已知的病毒和恶意软件，找到的就删除或隔离，并实时监控威胁；必须用最新的病毒库保持更新 |
| 磁盘碎片整理（defragmentation） | 重新组织磁硬盘，让每个文件的部分存放在连续的扇区里，减少读写头的移动，加快访问速度 |
| 磁盘内容分析 / 磁盘修复（disk contents analysis / disk repair） | 检查磁盘错误，报告存了什么、存在哪里，并尝试修复损坏的扇区或损坏的文件系统 |
| 文件压缩（file compression） | 减小文件（以及成组文件）的体积，使其占用更少存储空间、传输更快，需要时再还原成原样 |
| 备份软件（back-up software） | 为文件（或整块磁盘）制作副本，使数据在丢失、损坏或故障后能恢复；可以做完全备份或增量备份 |

要避免两个坑。第一，实用程序不是 OS 内核的一部分——它们是随 OS 一起提供的独立程序。第二，碎片整理只对磁硬盘有意义，因为文件会在盘片上散落；在 SSD 上数据块不会移动，所以整理没有好处，反而增加磨损。

## 3. 程序库和动态链接库

程序库（program library）是一批预先写好、预先编译好的代码，执行常见任务（数学函数、输入输出例程、窗口控件），开发者可以直接复用，而不必从头写。开发中的软件，绝大多数是把开发者自己写的代码和一个或多个程序库里的现成代码组合起来。

对开发者而言，好处就是复用的好处：

- 省时省力：常见函数不必重写。
- 省内存、省文件体积，因为库代码只编写和测试一次，可被多个程序共用。
- 可靠：库代码通常已被大量用户充分测试。
- 便于维护：库可以更新（比如修 bug），所有用到它的程序都受益。
- 让开发者能用别人写的代码，而不必理解或重现其中的每个细节。

动态链接库（Dynamic Link Library，DLL）是 Windows 系统中使用的一种特别的库。它的关键特性就在名字里——动态（dynamic）和链接（link）。库代码不是在编译时被复制进每个可执行文件（静态链接），而是程序只在运行时才与 DLL 链接（动态链接）。DLL 是一个独立文件，放在系统里，多个程序可以在运行时共用这一份。

DLL 的好处直接来自这一点：

- 可执行文件更小，因为库代码没有被复制进每个程序。
- 多个程序同时运行时节省内存，因为它们共用 RAM 中加载的那一份 DLL。
- DLL 可以更新（在系统上放一个新版本），所有用到它的程序自动获得新版本，不必逐个重新编译。
- 用不同语言编写的不同程序可以共用同一个 DLL。

代价是程序现在依赖于目标机器上存在这个 DLL 文件。如果 DLL 缺失或版本不对，程序就无法运行。

## 4. 语言翻译器：汇编器、编译器、解释器

源代码用高级语言或汇编语言写成；处理器只执行机器代码。语言翻译器把前者转换成后者。翻译器有三种，选哪种取决于源语言以及程序的用途。

- **汇编器（assembler）** 把汇编语言程序（每条机器指令对应一个助记符）翻译成机器代码。
- **编译器（compiler）** 把高级语言程序一次性翻译成机器代码，生成一个可执行文件，可日后运行。
- **解释器（interpreter）** 把高级语言程序逐条语句翻译并执行，不生成单独的可执行文件。

### 对比

| 特性 | 汇编器 | 编译器 | 解释器 |
|---|---|---|---|
| 源语言 | 汇编语言 | 高级语言 | 高级语言 |
| 输出 | 机器代码 | 一个完整的可执行文件 | 没有——直接运行程序 |
| 翻译单位 | 整个程序 | 整个程序 | 一次一条语句 / 一行 |
| 执行速度 | 快（只翻译一次） | 快（只翻译一次；经过优化） | 慢（每行每次运行都要重新翻译） |
| 分发 | 分发汇编后的程序 | 分发可执行文件；源代码保密 | 必须分发源代码和解释器 |
| 错误检测 | 汇编时发现错误 | 编译整个程序后一次性报出所有错误 | 运行时一次发现一个错误；遇到第一个错误就停止 |
| 调试 | 较难 | 较难（错误在最后才报出） | 较易（停在第一个错误，适合开发期） |

### 为给定场景论证编译器还是解释器

不要背固定答案；按场景的需要来选。

- **用编译器当** 程序要分发给用户、执行速度重要、源代码必须保密时。编译器生成一个可执行文件，跑得快，又不泄露源代码。
- **用解释器当** 程序还在开发、每条语句要立刻测试、或想要交互环境时。解释器停在第一个错误，使开发期定位和修复 bug 更快。
- **用汇编器当** 程序用汇编语言写成，也就是需要直接控制特定硬件、或要求最小最快的代码时。

典型的考题会给出一个情境——“老师在课堂上开发程序”、“公司卖一款文字处理软件”——然后问用哪种翻译器。把情境对到速度、保密性和调试需求上，再给出理由。

## 5. 部分编译加部分解释（Java）

有些语言既不是纯编译，也不是纯解释。Java（控制台模式）是 syllabus 的例子：它的源代码被部分编译，再被部分解释。

流程是：

1. Java 的源代码（`.java`）被 Java 编译器**编译**，但不是编译成任何特定处理器的机器代码，而是编译成一种中间形式，叫**字节码（bytecode）**（`.class`）。
2. 字节码是平台无关的：同一份字节码可以发到任何机器上。
3. 在目标机器上，**Java 虚拟机（JVM）** 读取字节码并**解释**它，逐条指令地在那台机器的处理器上执行。

所以两半分别是：一个编译步骤（源代码到字节码），生成可移植的中间代码；一个解释步骤（字节码到本机执行），在 JVM 里运行。好处是可移植性——一次编写，到处运行（只要那里有 JVM）。代价是解释比直接运行本机机器代码慢。

这就是为什么说 Java 是**部分编译、部分解释**，而不是“只编译”或“只解释”。第一步是编译，但产物是字节码而非机器代码；第二步是 JVM 的解释。

## 6. 集成开发环境（IDE）的功能

集成开发环境（Integrated Development Environment，IDE）是一个单一应用程序，把程序员写代码、检查代码、呈现代码和调试代码所需的工具整合在一起：编辑器、编译器或解释器、调试工具都集成在一个包里。syllabus 列出的功能按它们服务的开发阶段分组。

| 功能 | 开发阶段 | 做什么 |
|---|---|---|
| 上下文敏感提示（context-sensitive prompts） | 编写代码 | 编辑器根据程序员刚输入的内容提供补全或建议（例如列出一个对象的方法） |
| 动态语法检查（dynamic syntax checks） | 初步错误检测 | 编辑器在程序员输入时就按语言的语法规则标记语法错误，在编译之前 |
| 整齐打印（prettyprint） | 呈现 | 自动用一致的缩进和空格格式化代码，使其易读 |
| 展开和折叠代码块（expand and collapse code blocks） | 呈现 | 让程序员把函数或代码块的主体收起来，只显示头部，使大文件易于浏览 |
| 单步执行（single stepping） | 调试 | 一次运行一条语句，使程序员能观察每行之后发生了什么 |
| 断点（breakpoints） | 调试 | 标记程序会暂停的行，使程序员能在感兴趣的地方停下来检查状态 |
| 监视变量和表达式（报告窗口） | 调试 | 让程序员在程序运行时跟踪所选变量和表达式的值，显示在报告窗口里 |

要记住的模式是四个阶段：**编写、错误检测、呈现、调试**。把每个功能对应到它服务的阶段，回答 IDE 题时先报阶段、再说功能即可。

## 做题顺序

无论系统软件的哪类题，都可以按这个顺序走。

1. 先判断题目问的是 OS、实用程序、库 / DLL、翻译器，还是 IDE。它们是不同的东西，答案必须对得上。
2. OS 题先点明是哪项管理任务（内存、文件、安全、硬件、进程），再描述它做什么。不要把实用程序写在这里。
3. DLL 的好处从“动态链接”出发，顺着推下去：可执行文件更小、内存共用、可更新、可跨语言。
4. 编译器还是解释器，从速度、分发 / 保密、调试三方面决定，再按场景给出理由。
5. Java 题永远写两步：源代码到字节码（编译），字节码在 JVM 中本机执行（解释）。
6. IDE 功能题先报开发阶段（编写 / 错误检测 / 呈现 / 调试），再说功能。

## 常见错误

- 把实用程序（碎片整理、病毒检查）当成 OS 内核的一部分。实用程序是随 OS 提供的独立程序。
- 以为 SSD 也需要碎片整理。SSD 上文件不会散落，整理没有好处，反而增加磨损。
- 混淆静态链接和动态链接。静态链接把库代码复制进可执行文件；动态链接（DLL）在运行时链接到一个共用文件。
- 说 Java 是“只编译”或“只解释”。它先被部分编译成字节码，再由 JVM 部分解释。
- 混淆编译器和解释器的输出。编译器生成可执行文件；解释器什么都不生成，直接运行程序。
- 忘了解释器每次运行时都要重新翻译每条语句，这就是它慢的原因。
- 列 IDE 功能时不和开发阶段对应起来。
- 答“为什么要用 DLL”时只说“省内存”。要给出动态链接的后果：可执行文件更小、内存共用、可更新、跨语言。

## 快速自查

下面这些问题不用看笔记也能答上来，就可以进入下一节。

1. 为什么计算机系统需要操作系统？给出两条理由。
2. 说出 OS 执行的五项管理任务，以及每项管理什么。
3. 说明下列实用程序各做什么：磁盘格式化、病毒检查、碎片整理、备份软件。
4. 为什么碎片整理对 SSD 没用？
5. 什么是程序库？给出开发者使用它的两个好处。
6. 什么是动态链接？两个程序同时运行时，DLL 怎样节省内存？
7. 说出分发一个使用 DLL 的程序的一个好处和一个坏处。
8. 从输出、速度、错误检测三方面比较编译器和解释器。
9. 为一家卖文字处理软件的公司论证使用编译器。
10. 描述使 Java 成为部分编译、部分解释的那两个步骤。
11. 为四个开发阶段各举一个 IDE 功能：编写、错误检测、呈现、调试。
