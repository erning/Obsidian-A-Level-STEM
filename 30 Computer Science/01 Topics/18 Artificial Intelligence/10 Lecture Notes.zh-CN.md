## 1. AI 中的图：用途与结构

**图（graph）** 由 **节点（node，也称 vertex）** 和连接节点的 **边（edge，也称 link 或 arc）** 组成。每条边可以带一个 **权重（weight）**，一个表示走这条边代价的数字——距离、时间、油耗、难度，任何你想最小化的量。在 AI 中，图是一种把问题建模出来的手段，使"求解"变成"在图上搜索一条好路径"。

两种典型用途：

- **地图寻路**——节点是地点，带权重的边是它们之间的道路；搜索找出最短或最省的路线。
- **状态空间搜索**——每个节点是问题的一个状态（棋局、谜题的某种排列），每条边是一步合法的走法，搜索寻找从起始状态到目标状态的路径。

一个带权重的小图：

```text
          4
    A ---------- B
    | \          | \
   2|  \3        |  \1
    |   \        |   \
    C    D ----- E    F
     \3 |  \2   |1 /
      \ |   \   | /
       G ---- H
          5
```

节点：A、B、C、D、E、F、G、H。一条如 `A--4--B` 的边表示"可以在 A 与 B 之间通行，代价为 4"。搜索问题是：从某个节点（比如 A）出发，找到通往目标节点（比如 H）的最佳路径。"最佳"通常指总权重最低。

图可以是 **有向的**（边只单向通行）或 **无向的**（边双向通行，如上面的地图）。结构相同，只是通行方向不同。

## 2. 搜索算法：Dijkstra 与 A\*

Dijkstra 算法和 A\* 都在带权图上寻找从起始节点到目标节点的路径。它们共享"前沿（frontier）"和"暂定距离表"的思想：前沿是那些最短代价尚未最终敲定的节点，暂定距离记录到目前为止从起点到各节点的已知最短代价。

### Dijkstra 算法

**Dijkstra 算法**保证找出从起始节点到其他每个节点的**最短路径**（在权重非负的前提下）。它反复取出前沿中暂定距离最小的节点，将其标记为**最终确定**，再用它去松弛（改进）其邻居的暂定距离。

高层步骤：

1. 把起始节点的距离设为 0，其他全部设为无穷大。把起点放入前沿。
2. 取出前沿中**暂定距离最小**的节点。它现在**最终确定**——其最短距离锁定。
3. 对该确定节点的每个邻居，计算"当前代价 + 边权重"。如果比邻居存储的暂定距离更优，就更新它，并记录回溯路径（经由这个确定节点）。
4. 重复步骤 2-3，直到目标被确定（或所有可达节点都被确定）。
5. 从目标沿记录的回溯指针一路回到起点，再反转，即得路径。

因为每个节点只在真正最小时才被确定，Dijkstra 从不修改已确定的值——它是**最优的**。

### Dijkstra 的手工追踪

在第 1 节的图上求 A → H 的最短路径。为使追踪清晰，采用这组确定的边：

```text
A-B 4   A-C 2   A-D 3   C-D 1   D-E 2   E-H 1   C-G 4   G-H 3
```

起点 A，目标 H。前沿演化时的暂定距离如下，括号内表示已确定。

| 步骤 | 确定的节点 | 前沿（节点: 暂定距离） | 说明 |
|---|---|---|---|
| 0 | — | A:0 | 其余均为 ∞ |
| 1 | A (0) | B:4, C:2, D:3 | 最小是 C |
| 2 | C (2) | D:3, B:4, G:6 | A→C→D = 3 与 A→D = 3 相等，取 3 |
| 3 | D (3) | B:4, E:5, G:6 | A→C→D→E = 2+1+2 = 5 |
| 4 | B (4) | E:5, G:6 | B 的其他邻居更差 |
| 5 | E (5) | H:6, G:6 | E→H 给出 5+1 = 6 |
| 6 | H (6) | G:6 | 目标确定——停止 |

A → H 的最短距离为 **6**。路径从 H 回溯：H ← E ← D ← C ← A，即 **A → C → D → E → H**。

### A\* 算法

**A\***（A-star）通过加入一个**启发函数（heuristic）**——对每个节点到目标剩余代价的估计——更快地找到通往**单个目标**的路径。它不再选择暂定距离最小的前沿节点，而是选择 `f = g + h` 最小的节点，其中：

- **g** 是从起点到该节点的精确代价（与 Dijkstra 相同）；
- **h** 是该节点到目标剩余代价的启发式估计。

只要启发函数**从不高估**真实剩余代价（即**可采纳**），A\* 仍然找到最优路径，但探索的节点少得多，因为它被拉向目标，而不是向四面八方均匀铺开。地图上的一个 h 的例子是到目的地的直线距离"乌鸦飞"，它不可能超过真实道路距离。

### 对照

| 方面 | Dijkstra | A\* |
|---|---|---|
| 选择规则 | 最小暂定距离 `g` | 最小 `f = g + h` |
| 是否用启发函数 | 否 | 是 |
| 探索方式 | 四面八方均匀 | 被拉向目标 |
| 探索节点数 | 多（铺得广） | 较少（聚焦），当 h 良好时 |
| 保证最短路径？ | 总是（权重非负） | 是，当 h 可采纳（从不高估）时 |
| 最适合 | 到**所有**节点的最短路径；无目标估计可用 | 到**单个目标**的路径；有良好启发函数可用 |

权衡在于：A\* 的好坏取决于启发函数。当处处 `h = 0` 时，它就退化回 Dijkstra。

## 3. 人工神经网络

**人工神经网络（Artificial Neural Network，ANN）** 是一种受大脑启发的模型。它由排成层的**神经元（neuron，也称 node 或 unit）** 构成：

- 一个**输入层**接收数据（每个输入值对应一个神经元）。
- 一个或多个**隐藏层**对数据进行变换。
- 一个**输出层**给出答案（一个类别、一个数值、一个决策）。

神经元之间的每条连接带有一个**权重（weight）**——一个决定一个神经元输出对下一个神经元影响强度的数字。每个神经元把加权输入求和，加上一个**偏置（bias）**，再通过一个**激活函数（activation function）**（如阶跃、sigmoid 或 ReLU）决定其输出。

```text
输入层     隐藏层       输出层

 (x1) ----\
           \-->(h1)--\
 (x2) ----/--        \-->(y)
           \-->(h2)--/
 (x3) ----/    ...   /
```

关键在于：**权重是从数据中学习来的**，而不是手工编出来的。给定大量输入样本和正确输出，训练过程每次微调所有权重，直到网络的输出与样本吻合。这正是**机器学习**在难题上变得实用的原因：图像识别、语音、翻译这类问题，写出显式规则根本不可能，ANN 通过调节成千上万乃至上百万个权重，自己把规则学出来。

## 4. 机器学习、深度学习、强化学习

这三个词描述的是系统如何改进，它们彼此嵌套与重叠。

| 方法 | 是什么 | 学习信号 | 为什么用它 | 典型用途 |
|---|---|---|---|---|
| **机器学习（ML）** | 从数据中学习模式、而非遵循固定规则的系统 | 训练数据（样本） | 任务太复杂无法手写代码；模式藏在大量数据中 | 垃圾邮件过滤、价格预测、推荐 |
| **深度学习（DL）** | 使用**深层** ANN——许多隐藏层——的 ML | 训练数据，同 ML | 富含结构（图像、声音、语言）的任务，需要多层来学习层次化特征 | 图像识别、机器翻译、语音转文字 |
| **强化学习（RL）** | 一个**智能体（agent）** 在**环境**中行动，从**奖励**或**惩罚**中学习 | 奖励信号（不是带标签的样本） | 序贯决策，每一步没有唯一"正确答案"，靠试错学习 | 博弈（国际象棋、围棋）、机器人、导航、控制 |

它们之间的关系：

- **DL 是 ML 的子集**——每个深度学习系统都是一个机器学习系统，只不过它用的是深层神经网络。
- **RL 是另一种学习信号**——智能体不被给予带标签的样本，而是去探索，从动作赢得的奖励中学习。RL 智能体内部可以包含一个神经网络（如深度强化学习），但它的学习来自奖励，而非固定的数据集。

## 5. 机器学习的类别：监督学习与无监督学习

在 ML 内部，教学大纲点名的两个类别，区别在于训练数据是否带标签。

| 方面 | 监督学习 | 无监督学习 |
|---|---|---|
| 训练数据 | **有标签**——每个样本都附带正确答案 | **无标签**——只有输入，没有答案 |
| 目标 | 学习从输入到已知输出的映射 | 发现数据中隐藏的结构或分组 |
| 产出 | 一个分类器（类别）或一个回归模型（数值） | 簇、分组或降维后的数据 |
| 评估 | 把预测与已知标签比较 | 较难——没有"正确答案"可比 |
| 示例任务 | 垃圾邮件与否、识别数字、预测房价 | 客户分群、把相似新闻归到一起、异常检测 |

经验法则：如果你能说出"这个样本的正确答案是 X"，就是监督学习；如果你只有数据，必须问"哪些归在一起"，就是无监督学习。

## 6. 误差反向传播与回归

### 误差反向传播

**误差反向传播（back propagation of errors，常简称 backprop）** 是神经网络在监督训练中调整权重的方法。它从输出端的误差开始，向后传播。

高层步骤：

1. **前向传播**——把一个训练样本送入网络，记录网络实际产生的输出。
2. **计算误差**——把实际输出与正确（目标）输出用误差函数（如差值或平方误差）比较。
3. **向后传播误差**——从输出层向隐藏层逐层回退，算出每个权重对误差负了多少责任。
4. **调整权重**——按减小误差的方向，把每个权重微调一小步（步长由**学习率**控制）。
5. 在大量样本上反复迭代，直到网络的误差足够小。

直观地说：反向传播把输出误差的"责任"分摊给每个权重，然后微调权重，使下次同样的输入产生更接近目标的输出。（教学大纲不要求代码，只需上述理解。）

### 回归方法

机器学习中的**回归（regression）** 指**预测一个连续值**——尺度上的一个数——而非一个类别。这与**分类（classification）** 相对，后者预测离散的标签。

最简单的形式是**线性回归（linear regression）**：用一条直线拟合数据点，使这条线能从输入最好地预测输出。给定一组输入-输出对，直线

$$y = mx + c$$

的选取要使每个真实点与直线之间的总误差（常为平方差之和）最小。一旦学成，这条线就能对任何新的 x 预测 y——例如由房屋面积预测房价。

更复杂的回归（多项式回归、多元回归）拟合曲线或曲面，但思想一致：通过最小化训练数据上的预测误差，学习一个从输入到连续输出的函数。

## 做题顺序

1. 遇到图结构的题，点明三部分（节点、边、权重），说明在具体问题中各代表什么，并各举一个例子。
2. 遇到 Dijkstra 的题，保持前沿表整洁：列里写暂定距离，已确定节点做好标记，下一步永远取最小的。
3. 遇到 A\* 的题，写出公式 `f = g + h`，定义各量，并强调 h 绝不能高估真实剩余代价。
4. 比较 Dijkstra 与 A\* 时，从保证出发搭表（两者都给最短路径，但 A\* 用启发函数聚焦搜索）。
5. 遇到 ANN 的题，画出输入 → 隐藏 → 输出层，标出权重，并说明权重是**学来的**，不是编出来的。
6. ML、DL、RL 要按嵌套放置：DL ⊂ ML；RL 是另一种基于奖励的信号。
7. 监督与无监督，先判断数据是否有标签——这一个事实就决定了类别。
8. 反向传播先走前向（算输出），再走后向（传播误差、调权重），按此顺序。
9. 回归题先说明输出是连续值，再给出直线的直观理解，最后才谈细节。

## 常见错误

- 描述图时漏掉权重，或忘记权重代表代价。
- 以为 Dijkstra 需要目标——它找的是到**所有**节点的最短路径，目标只是停下来的地方。
- 说 A\* 不保证最优路径。只要启发函数可采纳，它就保证最优。
- 用了不可采纳的启发函数（会高估）还声称最优。
- 把激活函数与权重混为一谈。权重缩放输入；激活函数决定神经元的输出。
- 把深度学习当成独立于机器学习之外的东西。DL 是 ML 的子集。
- 把强化学习与监督学习混淆：RL 没有带标签的样本，只有奖励信号。
- 没先检查数据是否有标签，就把监督与无监督弄混。
- 描述反向传播时只说"调权重提高准确率"，却不说它是**从误差向后**进行的。
- 把回归说成"分类"。回归预测连续值；分类预测类别。
