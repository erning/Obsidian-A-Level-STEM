# Communication 中文讲义

一台计算机只能自己计算，联网的计算机却能共享、请求、协作。这一节讲的是分层的连接：底层是物理介质和硬件，把机器用线缆或无线电波连成本地网络；中间层是局域网的拓扑结构和归属模型，决定网络怎么组织；顶层是覆盖全球互联网的寻址和命名机制，把无数网络串成一张大网，并在其上承载万维网等服务。把这三层叠起来看，细节就不会乱。

## 1. 为什么要组网：LAN 和 WAN

把若干台计算机连起来，就构成网络。组网带来的好处很直接：**资源共享**（一台打印机或文件存储服务多个用户）、**通信**（邮件、即时消息、网络电话）、**集中管理与备份**数据、通过一个网关**共享上网**，以及**便于扩展**（新机器直接接入）。

按覆盖范围，网络先分成两类。

局域网（LAN）覆盖较小的地理范围，比如一栋楼、一所学校或一个办公室。它通常**由使用它的机构自己拥有和维护**，数据率较高，误码率较低，因为距离短，可以用质量好的线缆和交换机，一般不需要外部电信线路。

广域网（WAN）覆盖**较大的地理范围**，可以是城市、国家甚至全球，把各地的局域网和单机连起来。WAN 几乎总要**借助第三方电信链路**（电话线、专线、光纤骨干、卫星、蜂窝网络），数据率通常比 LAN 低，误码率较高，连接硬件归服务提供商所有，不归最终用户。互联网就是最有名的 WAN。

| 方面 | LAN | WAN |
| --- | --- | --- |
| 覆盖范围 | 一栋楼/一个站点 | 城市到全球 |
| 归属 | 通常归机构自己 | 第三方电信提供商 |
| 典型速率 | 高 | 较低 |
| 误码率 | 低 | 较高 |
| 连接介质 | 内部铜线/光纤/WiFi | PSTN、专线、卫星、蜂窝 |

## 2. 网络模型与客户端类型

### 客户端-服务器 与 对等网络

客户端-服务器（client-server）模型里，有一台或多台专门的**服务器**提供服务（文件存储、打印、邮件、网站、身份认证），其余机器作为**客户端**请求这些服务。角色是固定的：服务器是权威，客户端是消费者。

好处：资源、数据和备份集中管理；安全策略和用户账号集中在一处；性能好，因为繁重的工作由强大的服务器承担；扩展方便，升级服务器即可。缺点：需要专门的 IT 人员；服务器是**单点故障**，一旦宕机所有人都无法工作；购置和维护成本高。

对等网络（peer-to-peer，P2P）里每台计算机**地位平等**，每个节点既能当客户端也能当服务器，把自己的文件、打印机或算力直接分享给其他节点，没有中央权威。

好处：便宜、搭建简单，不需要专门的服务器和 IT 人员；健壮，因为失去任何一个节点都不会让整个网络瘫痪；增加节点容易。缺点：安全弱，没有集中管控；备份只能靠每台机器自己做；节点越多性能越差；查找文件和管理用户更麻烦。

为具体场景选择模型时这样判断：银行、学校这类需要集中、安全、规范管理数据的，选客户端-服务器；家庭网络或几个人共享少量文件、或者文件分享社区这种必须容忍节点随时加入离开的，选对等网络。

### 瘦客户端 与 胖客户端

这两种说法描述的是客户端自己做多少处理。

瘦客户端（thin-client）主要**依赖强大的中央服务器**，客户端本身很轻（存储少、CPU 不强），主要做输入和显示，处理和数据都在服务器上。

胖客户端（thick-client）把大部分处理放在**本地 CPU 和存储**上完成，只在需要时才向服务器请求数据。

看重集中管控、维护简便和廉价终端的场景（呼叫中心、学校机房、高安全环境）适合瘦客户端；需要本地算力和离线工作的用户（开发者、设计师、游戏）适合胖客户端。瘦客户端降低了单机成本和管理负担，但严重依赖快速可靠的网络和强劲的服务器；胖客户端单机更贵、更难统一打补丁，但即使网络慢也能正常工作。

## 3. 拓扑结构：总线型、星型、网状、混合型

拓扑结构（topology）是节点之间链路的物理或逻辑排列方式，决定了成本、性能以及某部件故障时会发生什么。

| 拓扑 | 成本/布线 | 性能 | 容错 |
| --- | --- | --- | --- |
| 总线型（bus） | 最低，一根主干线 | 节点越多越差，冲突频繁 | 差，主干线或终端器损坏会使全网瘫痪 |
| 星型（star） | 中等，每个节点一根线到中央设备 | 好，中央交换机隔离流量，冲突少 | 节点侧好（一根线坏只影响一台机器）；中央设备是单点故障 |
| 网状（mesh） | 最高，大量点对点链路 | 优秀，多条路径，无单一瓶颈，便于增加节点 | 极高，流量可绕过失效链路 |
| 混合型（hybrid） | 视组合而定 | 可按区域需要调整 | 继承各组成部分的优缺点 |

### 数据包在各拓扑中怎么走

总线型：所有节点都接到同一根主干线上。主机 A 发出的数据包广播到总线上，每个节点都能看到，但只有地址匹配的主机才留下。两端各有一个终端器吸收信号，防止反射。两台主机同时发送就会**冲突**。

星型：每个节点都有自己的线接到中央交换机。A 发的包先到交换机，交换机只把它转发到目标端口。某条线坏了只影响那一台机器。

网状：每个节点都和其他多个节点有点对点链路。一个包从源到目标可以走很多条路，所以某条链路断了，包可以**改走别的路径**。这种冗余是网状的核心优势，但需要的线缆最多。

混合型：比如若干个星型网络用总线或网状骨干连起来。包在星型内部经由中央交换机传递，跨星型时走骨干。

为场景选拓扑：廉价、临时的小型机房可以接受总线型；典型办公室局域网用星型，平衡了成本和可靠性；医院、军队这种必须持续在线的选网状；需求多样的大型机构用混合型。

## 4. 云计算

云计算（cloud computing）是指使用托管在远程服务器上的计算资源（存储、算力、应用），通过网络交付，而不是在本地运行。硬件由提供商拥有和维护，用户按使用量付费。

公有云（public cloud）：服务和基础设施归第三方提供商（如 AWS、Azure）所有，通过互联网在大量客户之间共享。便宜，无需自购硬件，可即时扩展，但数据要离开本机构，且依赖提供商和网络质量。

私有云（private cloud）：基础设施只服务于单一机构，可以内部部署，也可以由提供商托管但不与他人共享。更安全、更可控，适合敏感数据，但成本更高，需要内部专业技术。

好处：前期成本低（无需购买硬件），按量付费，易于扩展，随处访问，维护、安全补丁和备份都由提供商负责。缺点：需要可靠的网络连接；数据存放在外部带来隐私和安全顾虑；依赖提供商；订阅费用会持续累积；可能被锁定在某一厂商。

## 5. 传输介质与局域网硬件

### 有线与无线

有线网络使用实体线缆。速率高、可靠、安全（信号不易被截获），不受天气和干扰影响，但布线成本高，设备不能移动，在复杂地形或楼宇间不便于布线。

无线网络通过空气中的电磁波传输。设备可**移动**，无需布线、部署便宜，但信号会受干扰、衰减，存在安全风险（范围内的任何人都能监听），速率通常比有线低且更不稳定。

### 各种介质的特性

| 介质 | 传输方式 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 铜线（双绞线、同轴） | 电信号 | 便宜、普及、易安装 | 易受电干扰和窃听，信号随距离衰减 |
| 光纤 | 玻璃中的光脉冲 | 带宽极高，不受电磁干扰，难以窃听，可远距离传输 | 昂贵、易碎，需要专业安装和设备 |
| 无线电波/WiFi | 空气中的无线电波 | 可移动、无需布线、易增加设备 | 范围有限，受干扰和安全风险影响，速率低于有线 |
| 微波 | 高频无线电，视距传播 | 容量大、可远距离（塔之间） | 需要视距，受天气和障碍物影响 |
| 卫星 | 经轨道卫星的微波链路 | 覆盖范围极广，含偏远地区和海洋，适合电视、GPS、通信 | 昂贵，信号延迟明显（尤其静止轨道），恶劣天气下质量下降 |

### 局域网硬件

| 设备 | 在局域网中的作用 |
| --- | --- |
| 交换机（switch） | 连接节点，读取每个帧的目标 MAC 地址，只转发到正确端口，从而减少冲突 |
| 服务器（server） | 强大的计算机，为客户端提供共享服务（文件、打印、认证、邮件） |
| 网卡（NIC） | 把设备接入有线网络，处理物理信号，并给设备分配 MAC 地址 |
| 无线网卡（WNIC） | 对应的无线版本，把设备接入 WiFi 网络 |
| 无线接入点（WAP） | 无线基站，让 WiFi 设备接入有线局域网，桥接无线和有线网段 |
| 线缆（cables） | 连接设备的物理介质（铜线或光纤） |
| 网桥（bridge） | 连接两个相似网段，按 MAC 地址过滤流量，使每个网段只承载自己的帧 |
| 中继器（repeater） | 清理并再生衰减的信号，使其能传得更远而不失真 |

### 路由器

路由器（router）工作在网络层，连接**不同的网络**（比如把家庭局域网连到互联网）。它的作用是读取每个数据包的 IP 地址，查询路由表，选择**最佳路径**把包转发到目标网络。在家庭网络里，路由器通常还充当到 ISP 的网关，在私有和公有 IP 之间做**网络地址转换（NAT）**，并可能内置交换机和 WAP。因为路由器按 IP 地址在不同网络之间路由，它比只在单一局域网内按 MAC 地址工作的交换机更强大。

## 6. Ethernet 与 CSMA/CD

Ethernet 是有线局域网的主流技术族，规定了帧格式、信号方式和访问方法，用于基于线缆和交换机的网络。Ethernet 帧带有源和目标 MAC 地址。在早期的共享介质 Ethernet（总线型或基于集线器的星型）中，多台主机共用一条信道，因此可能冲突。

CSMA/CD（载波监听多路访问/冲突检测）是让多台主机共享这条信道的访问方法。

1. **载波监听**：发送前先听介质。空闲就发，有流量就等。
2. **多路访问**：所有主机共享同一介质，任何一台都可以发送。
3. **冲突检测**：发送时主机持续比较自己发出的信号和线上看到的信号。如果不一致，说明发生了**冲突**——另一台主机同时在发，两个信号互相干扰。
4. **避免/退避**：检测到冲突后，主机发送一个**拥塞信号**让所有站点知晓，然后**停止发送**，等待一个**随机退避时间**再重试。随机延迟使两台主机不太可能再次同时发送，这就是冲突被**检测**之后再被**避免**的机制。

现代交换式 Ethernet 基本消除了冲突（每个交换机端口是独立的冲突域），所以 CSMA/CD 主要对共享介质和历史理解有意义。

## 7. 比特流

比特流（bit streaming）是把音频或视频数据通过网络连续送达，使播放能在整个文件下载完之前就开始。播放器先缓冲几秒数据，然后边收边放。

实时流（real-time，直播）：内容随发生随发送（体育直播、视频通话）。观众不能暂停等待缓冲；如果连接跟不上，画质下降或卡顿。低延迟比完美画质更重要。

按需流（on-demand）：内容是预先录制、存放在服务器上的（流媒体上的电影）。播放器可以提前缓冲，所以稍慢的连接仍能流畅播放，只是开始延迟更长。

媒体的比特率和宽带速率必须匹配。如果媒体比特率超过可用带宽，流就必须频繁缓冲或降到更低分辨率。高清视频需要的比特率远高于音频或低分辨率视频，所以宽带速率直接限制了用户能流畅观看的画质。

## 8. 互联网与万维网，以及互联网硬件

互联网（internet）是**全球物理基础设施**：数以百万计的互联网络、路由器、线缆、光纤骨干和无线链路，在世界范围的设备之间承载数据。它是介质。

万维网（WWW）是**运行在互联网之上的服务**。它是用 URL 标识、用超链接相连的**网页和资源的集合**，通过 HTTP 等协议访问，用浏览器查看。其他服务（邮件、文件传输、游戏、即时消息）也运行在互联网上，但不属于万维网。

一个有用的判别：邮件在互联网上但不属于万维网；用浏览器看的网页两者都是。互联网可以没有万维网而存在（事实上早年如此），但万维网离不开互联网。

### 互联网硬件

| 硬件 | 作用 |
| --- | --- |
| 调制解调器（modem） | 把数字数据和能在电话线/有线电视线上传输的形式互相转换，使计算机能使用模拟电信链路 |
| PSTN（公共交换电话网） | 传统公共电话线网络，最初为语音设计，常用于承载拨号或 DSL 上网流量 |
| 专线（dedicated/leased line） | 为单一客户保留的永久、高质量、高带宽连接，适合需要稳定永在线的企业 |
| 蜂窝网络（cell phone network） | 移动电话基础设施（基站塔），为智能手机和移动热点承载数据，无需固定布线即可上网 |

## 9. IP 寻址、子网划分、公有/私有与静态/动态

互联网上的每台设备都需要一个 IP 地址，数据包才能路由到它。IP 地址通过网卡（NIC/WNIC）关联到设备，地址被配置到网络接口上（手动或由服务器分配）；一台有多个接口的设备可以有多个地址。

### IPv4 与 IPv6

| 特性 | IPv4 | IPv6 |
| --- | --- | --- |
| 长度 | **32 位**，写成四个十进制八位组（如 `192.168.0.10`） | **128 位**，写成八组十六进制（如 `2001:0db8::1`） |
| 地址空间 | 约 $4.3 \times 10^9$ 个地址，现已耗尽 | 约 $3.4 \times 10^{38}$ 个地址，几乎无限 |
| 提出原因 | 原始方案 | 为解决 IPv4 枯竭并简化路由 |
| 可读性 | 对人友好 | 更长但更有结构 |

### 子网划分

子网划分（subnetting）把一个大网络拆成若干个更小的逻辑子网。IPv4 地址分两部分：**网络前缀**（标识网络）和**主机部分**（标识网络内的设备）。分界点用子网掩码（如 `255.255.255.0`）或 CIDR 长度（如 `/24`）表示。子网划分让机构能按物理或部门布局切分地址空间，提高安全性（隔离不同机器组），并减少每个子网的广播流量。例如 `192.168.1.0/24` 把前三段用作网络，最后一段用于最多 254 台主机。

### 公有 IP 与私有 IP（安全维度）

| 类型 | 用途 | 可路由性 |
| --- | --- | --- |
| 公有 IP（public） | 在全球互联网上唯一标识设备 | 可在互联网上路由 |
| 私有 IP（private） | 只在局域网内部使用（如 `192.168.x.x`、`10.x.x.x`） | 在公网上**不可**路由 |

安全含义：私有地址有一定保护作用，因为它们不能从互联网直接访问；执行**网络地址转换（NAT）**的路由器把许多私有地址映射到一个公有地址，隐藏了内部结构并节省公有地址。公有地址是暴露的，因此需要防火墙等保护。

### 静态 IP 与动态 IP（分配维度）

| 类型 | 获取方式 | 典型用途 |
| --- | --- | --- |
| 静态（static） | 固定，手动设置，永不变 | 服务器、打印机、必须始终在同一地址找到的网站主机 |
| 动态（dynamic） | 设备接入时由 DHCP 服务器自动分配，可能变化 | 大多数家庭和办公设备 |

注意，公有/私有与静态/动态是**两个相互独立的维度**：一个地址可以是公有静态（网站服务器）、公有动态（ISP 地址轮换的家庭连接）、私有静态（内部固定服务器）或私有动态（接入办公室 WiFi 的笔记本）。考试时千万不要把它们混在一起。

## 10. URL 与 DNS

URL（统一资源定位符）是万维网上某个资源的地址。它的各部分分别指明访问方式、主机和资源路径：

```
https://www.example.com/courses/page.html
  \_/   \______________/\________________/
协议         主机               路径
```

协议（`https`）是获取资源用的协议；主机（`www.example.com`）是服务器的域名，是便于人阅读的标签；路径（`/courses/page.html`）是资源在该服务器上的位置。

但互联网是按 **IP 地址**路由数据包的，不是按名字，所以必须先把域名解析成 IP 地址。这就是 DNS（域名系统）的工作：它是一个全球分布式数据库，把域名映射到 IP 地址。

### DNS 解析，逐步说明

1. 用户在浏览器输入 URL，浏览器提取出域名。
2. 浏览器向配置好的 DNS 解析器（通常由 ISP 或公共解析服务运行）询问该域名的 IP 地址。
3. 若解析器不知道，就先查询**根域名服务器**，根服务器把它指向对应顶级域名（TLD）服务器（如 `.com`、`.org`）。
4. TLD 服务器再把它指向该域名的**权威域名服务器**，那里保存着官方记录。
5. 权威服务器返回该域名的 IP 地址。
6. 解析器把结果缓存起来，并把 IP 地址交回浏览器。
7. 浏览器据此 IP 地址建立连接，用 HTTP 请求资源。

正是 DNS 让万维网上便于人类阅读的名字，能在按 IP 编址的互联网上使用。

## 做题顺序

1. 遇到"描述这个网络"的题，先定位场景：是 LAN 还是 WAN，什么模型，什么拓扑，什么介质。
2. 遇到拓扑题，先画出两个指定主机之间一个数据包的实际路径，再据此判断成本、性能和容错。
3. 遇到 IP 题，把两个维度分开判断：这个地址是公有还是私有、是静态还是动态，然后分别说含义。
4. 遇到 DNS 题，按 浏览器 → 解析器 → 根 → TLD → 权威服务器 → 返回 的顺序走一遍。
5. 遇到 CSMA/CD，记住顺序：**先监听，再检测，然后随机退避**。

## 常见错误

- 把互联网和万维网说成同一个东西；万维网是互联网上的服务。
- 把公有/私有（可达性）和静态/动态（分配方式）混为一谈；它们是两个独立维度。
- 把 CSMA/CD 说成只会检测冲突；它还必须通过随机退避**避免**冲突。
- 把路由器说成"按 MAC 地址交换"；那是交换机。路由器按 IP 在不同网络之间路由。
- 忘记私有 IP 地址在公网上不可路由，要靠 NAT。
- 把瘦客户端理解成"完全没有客户端"；它仍有输入输出硬件，只是处理能力弱。
- 把光纤（光，不受电干扰）和铜线（电，易受干扰）的特性搞混。
- 说微波或卫星不需要视距；两者都受障碍物和天气影响。
