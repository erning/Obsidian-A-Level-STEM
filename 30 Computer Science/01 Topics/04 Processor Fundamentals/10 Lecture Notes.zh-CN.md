# Processor Fundamentals 中文讲义

计算机之所以有用，是因为它能一条一条地执行预先存好的程序。这一节讲的就是做这件事的引擎：CPU 如何构造，如何反复地取指和执行指令，这些指令在汇编层面长什么样，以及它如何操作单个位去控制真实设备。把取指-执行周期和各寄存器的作用理清，追踪汇编程序就从玄学变成机械流程了。

## 1. 冯·诺依曼体系结构与 CPU 部件

**冯·诺依曼体系结构（Von Neumann model）** 是几乎所有现代 CPU 的设计思想：程序（指令）和它处理的数据都存放在同一个主存里。这个**存储程序（stored-program）**概念意味着，同一块硬件只要往内存里装不同内容，就能干完全不同的活。CPU 不需要为每个任务重新接线，只要读取一条新的指令序列、依次执行即可。由于指令和数据共用一个内存和一组总线，CPU 无法在同一瞬间既取下一条指令又取操作数——这有时被称为冯·诺依曼瓶颈，但设计的简洁性带来的好处远大于这点代价。

CPU 内部要按作用记住四个部件。

- **ALU（算术逻辑单元，Arithmetic Logic Unit）。** 负责计算的部件。它执行指令集要求的所有算术运算（加、减、自增、自减）和所有逻辑运算（AND、OR、XOR、移位、比较）。
- **CU（控制单元，Control Unit）。** 负责指挥的部件。它译码存放在 CIR 中的指令，并发送控制信号，告诉其他每一个部件——ALU、内存、总线、寄存器——什么时候做什么。
- **系统时钟（system clock）。** 一块晶体振荡器，产生稳定的定时脉冲。取指-执行周期的每一步都与这些节拍同步；一个节拍是一个时钟周期，时钟频率以每秒周期数（Hz、GHz）计量。
- **IAS（即时存储器，Immediate Access Store）。** 位于 CPU 内部的存储，也就是它的寄存器，处理器无需访问主存就能取用。IAS 存放 CPU 当前正在处理的那少数几个值。

## 2. 寄存器

**寄存器（register）** 是 CPU 内部极快的存储位置，由触发器构成。它们分为两类。

- **通用寄存器（general-purpose register）** 可以由程序员用于任何暂存任务。本指令集里只有一个：**累加器（ACC）**，所有运算都通过它进行。**变址寄存器（IX）** 另行描述：它可由程序员寻址，主要用于变址寻址和充当计数器。
- **专用寄存器（special-purpose register）** 在取指-执行周期中各有固定用途，不能挪作他用。

| 寄存器 | 缩写 | 作用 |
|---|---|---|
| 程序计数器 | PC | 存放下一条要取的指令的地址 |
| 内存数据寄存器 | MDR | 存放刚从内存读出或即将写入内存的数据（也叫内存缓冲寄存器） |
| 内存地址寄存器 | MAR | 存放当前正被读取或写入的内存地址 |
| 当前指令寄存器 | CIR | 存放正在被译码和执行的指令 |
| 累加器 | ACC | 唯一的通用工作寄存器；每个算术和逻辑结果都在这里产生 |
| 变址寄存器 | IX | 专用变址寄存器；存放变址寻址用的偏移量，常作循环计数器 |
| 状态寄存器 | — | 存放 ALU 设置的标志位，例如比较标志（CMP 的结果）、零标志、符号标志、溢出标志、进位标志 |

状态寄存器有时也叫标志寄存器。它的每一位记录上一次 ALU 运算的一个事实。对追踪汇编最关键的是**比较标志**：`CMP` 指令在两个值相等时把它置为 True（1），不等时置为 False（0），而 `JPE`/`JPN` 测试的正是这一个标志。

## 3. 总线、性能因素与端口

CPU 通过三条总线与内存和外设通信，每条传输不同性质的信号。

- **地址总线（address bus）。** 传送 CPU 要读或写的地址。在冯·诺依曼机器里通常是**单向**的（CPU 到内存）。
- **数据总线（data bus）。** 传送实际读出或写入的数据。它是**双向**的。
- **控制总线（control bus）。** 传送定时与命令信号，如读/写、时钟、中断、复位。它是双向的。

有五个因素影响处理器的实际速度。

| 因素 | 如何影响性能 |
|---|---|
| 处理器类型 | RISC 设计执行更简单、更快的单周期指令；CISC 设计使用更丰富、更慢的指令。每条指令所需周期数越少，单位时间完成的越多。 |
| 核心数 | 每个核可以跑自己的指令流，所以多核 CPU 能对不同的任务或线程做真正的并行。 |
| 总线宽度 | 数据总线越宽，每次传输的位数越多；地址总线越宽，能寻址的内存位置越多。两者都能减少所需的传输次数。 |
| 时钟速度 | 每秒周期数越多，每秒完成的取指-执行周期就越多，单位时间内完成的指令也越多。 |
| 缓存（cache） | CPU 与 RAM 之间的小块快速 SRAM，保存最近用过的数据和指令，从而降低 CPU 等内存的平均时间。 |

**端口（port）** 是外设与计算机相连的插座。

- **USB（通用串行总线，Universal Serial Bus）。** 一种通用串行端口，几乎能接任何外设——鼠标、键盘、存储、打印机。支持热插拔，能给小型设备供电，一个接口支持多种设备。
- **HDMI（高清晰度多媒体接口，High-Definition Multimedia Interface）。** 数字端口，向显示器或电视同时传送高质量视频和音频。
- **VGA（视频图形阵列，Video Graphics Array）。** 较早的、仅传视频的模拟端口，至今仍见于投影仪和老式显示器。

## 4. 取指-执行周期

CPU 永远重复着**取指-译码-执行周期（fetch-decode-execute cycle）**，一次一条指令。用寄存器传送记号表示（其中 `←` 表示"被装入"，`[MAR]` 表示"MAR 中地址所指的内存内容"）：

```text
--- 取指 FETCH ---
MAR  ← PC                 ; 下一条指令的地址复制到 MAR
MDR  ← [MAR]              ; 控制单元发出读内存信号;
                           ; 指令沿数据总线传回
CIR  ← MDR                ; 取到的指令保存下来供译码
PC   ← PC + 1             ; PC 前进，指向下一条指令

--- 译码 DECODE ---
CU    译码 CIR 中的操作码
MAR  ← CIR 的操作数地址        ; 仅当该指令需要内存操作数时
MDR  ← [MAR]                  ; 取操作数本身

--- 执行 EXECUTE（取决于操作码） ---
; 例如  ADD <address>:
ACC  ← ACC + MDR
; ALU 按需把标志位写入状态寄存器
```

有两个细节容易让学生栽跟头。第一，PC 是在取指阶段就加 1 的，所以当这条指令执行完时，PC 已经指向下一条。第二，任何需要内存操作数的指令都会在译码/执行阶段再触发一次内存读取——取指令和取操作数是两笔独立的总线事务。

## 5. 中断

**中断（interrupt）** 是告诉 CPU 某个事件需要立即处理的信号。中断原因分两类。

- **硬件中断** 来自 CPU 外部：某 I/O 设备完成传输、硬件定时器到点、按键被按下，或掉电预警。
- **软件中断**（异常/陷阱）来自正在运行的程序内部：除零、算术溢出，或显式的系统调用。

CPU 收到中断时不会半途停下。它会先完成当前这条取指-执行周期，然后运行一小段叫**中断服务程序（Interrupt Service Routine，ISR）**的代码来处理事件。完整过程是：

1. CPU 先把执行阶段的当前指令做完。
2. 它检测到有待处理的中断（这一检查发生在每个取指-执行周期结束时、取下一条指令之前）。
3. 它保存当前状态——至少保存 PC，通常还包括其他寄存器——好让被打断的程序能在原地恢复。
4. PC 装入 ISR 的地址，所以下一次取指运行的就是 ISR。
5. ISR 执行，为设备服务或处理异常。
6. ISR 末尾用一条中断返回指令恢复保存的寄存器和 PC，原程序像什么都没发生一样继续运行。

## 6. 汇编语言

### 机器码与两遍汇编器

在硬件层面，每条指令不过是一个二进制串——**机器码（machine code）**。**汇编语言（assembly language）** 是它的人类可读形式：每一行是一个短**助记符（mnemonic）**（如 `ADD`）加上操作数，每个助记符直接对应一条机器码指令。一个叫**汇编器（assembler）**的程序把汇编翻译成机器码。

翻译之所以要**两遍（two-pass）**，是因为程序可以跳转到一个尚未定义的标号——即**向前引用（forward reference）**。

- **第一遍（Pass 1）。** 汇编器按顺序扫描程序，为每条指令分配内存地址，并建立一张**符号表（symbol table）**，记录每个标号的地址。它也会报告重复定义的标号。
- **第二遍（Pass 2）。** 汇编器再扫描一遍，把每条指令翻译成机器码，每遇到标号就去符号表里查出地址替换掉。

没有两遍，单次向前扫描就无法解决跳到文件下方某标号的问题，因为那时那个标号的地址还不知道。

### 指令分组

| 分组 | 用途 | 示例 |
|---|---|---|
| 数据传送 | 在 ACC、IX 和内存之间移动数据 | `LDD`、`LDX`、`STO`、`MOV` |
| 输入/输出 | 从外设读或向外设送数据 | `IN`、`OUT` |
| 算术运算 | 做计算 | `ADD`、`SUB`、`INC`、`DEC` |
| 无条件控制转移 | 总是跳转 | `JMP` |
| 条件控制转移 | 依据比较标志跳转 | `JPE`、`JPN` |
| 比较 | 通过比较两个值来设置比较标志 | `CMP`、`CMI` |
| 位操作 | 按位逻辑与移位 | `AND`、`OR`、`XOR`、`LSL`、`LSR` |

### 寻址方式

**寻址方式（addressing mode）** 是 CPU 用来确定操作数位置的规则。下面的每种方式都用使用它的装载指令来示例。

| 方式 | 如何找到操作数 | 示例 | 效果 |
|---|---|---|---|
| 立即数寻址 | 操作数就是指令里写出的那个值 | `LDM #5` | ACC ← 5 |
| 直接寻址 | 操作数存放在指令给出的地址里 | `LDD 100` | ACC ← M[100] |
| 间接寻址 | 给出的地址里存放的是另一个地址，操作数在那里 | `LDI 100` | ACC ← M[M[100]] |
| 变址寻址 | 操作数在（给出的地址 + IX）处 | `LDX 100` | ACC ← M[100 + IX] |
| 相对寻址 | 地址以相对当前 PC 的偏移量给出 | `JMP +3`（概念） | 跳到 PC + 偏移 |

相对寻址用于短小的、与位置无关的跳转：目标地址是从程序恰好装载到哪里算出来的，所以同一段代码无需改动就能跑在任意地址。

### 完整指令集速查

操作数中，`#` 表示十进制值，`B` 表示二进制，`&` 表示十六进制。

| 助记符 | 操作数 | 动作 |
|---|---|---|
| `LDM` | `#n` | 把立即数 n 装入 ACC |
| `LDD` | `<address>` | 把地址处的内容装入 ACC（直接寻址） |
| `LDI` | `<address>` | 把"地址处所存地址"的内容装入 ACC（间接寻址） |
| `LDX` | `<address>` | 把（地址 + IX）处的内容装入 ACC（变址寻址） |
| `LDR` | `#n` | 把值 n 装入 IX |
| `MOV` | `IX` | 把 ACC 的内容复制到 IX |
| `STO` | `<address>` | 把 ACC 的内容存到地址处 |
| `ADD` | `<address>` 或 `#n` | 加到 ACC（内存内容或立即数） |
| `SUB` | `<address>` 或 `#n` | 从 ACC 减去 |
| `INC` | `<register>` | ACC 或 IX 自增 1 |
| `DEC` | `<register>` | ACC 或 IX 自减 1 |
| `JMP` | `<address>` | 无条件跳转到地址 |
| `CMP` | `<address>` 或 `#n` | 将 ACC 与操作数比较；相等则比较标志置 True |
| `CMI` | `<address>` | 将 ACC 与"地址处所存地址"的内容比较（间接比较） |
| `JPE` | `<address>` | 若比较标志为 True（相等）则跳转 |
| `JPN` | `<address>` | 若比较标志为 False（不等）则跳转 |
| `IN` | — | 键入一个字符，并把它的 ASCII 值存入 ACC |
| `OUT` | — | 输出 ASCII 值存放在 ACC 中的那个字符 |
| `AND` | `#n`/`Bn`/`&n` 或 `<address>` | ACC 与操作数按位与 |
| `OR` | `#n`/`Bn`/`&n` 或 `<address>` | ACC 与操作数按位或 |
| `XOR` | `#n`/`Bn`/`&n` 或 `<address>` | ACC 与操作数按位异或 |
| `LSL` | `#n` | ACC 逻辑左移 n 位 |
| `LSR` | `#n` | ACC 逻辑右移 n 位 |
| `END` | — | 停止执行 |

## 7. 汇编追踪示例：用累加做乘法

下面的程序用"每有一个乘数单位就把被乘数加一次"的办法做乘法。被乘数放在地址 100，乘数（作计数用）放在地址 101，运行中的乘积保存在地址 200，地址 201 作保存计数用的临时位置。设 M[100] = 5、M[101] = 2，结果应为 10。

```asm
        LDD 101         ; ACC = 乘数（计数）
LOOP:   CMP #0          ; 计数是否为 0?
        JPE Done        ; 若是, 乘法完成
        DEC ACC         ; 计数 - 1
        STO 201         ; 把计数存到 M[201]
        LDD 200         ; ACC = 当前乘积
        ADD 100         ; ACC = 乘积 + 被乘数
        STO 200         ; 存回新乘积
        LDD 201         ; ACC = 已保存的计数
        JMP LOOP        ; 回去再测
Done:   LDD 200         ; ACC = 最终乘积
        OUT             ; 输出乘积
        END
```

窍门在于 ACC 是唯一的工作寄存器，所以乘积只能住在内存（地址 200）。每一轮迭代：先在 ACC 里把计数减 1，存到 201；再把乘积调入 ACC，加上被乘数，存回乘积；最后把保存的计数重新装入 ACC，以便再次测试循环。

设 M[100] = 5、M[101] = 2、M[200] = 0，走一遍：

| 步 | 指令 | ACC | M[200] | M[201] | 标志 | 说明 |
|---|---|---|---|---|---|---|
| 1 | `LDD 101` | 2 | 0 | – | – | 装入计数 |
| 2 | `CMP #0` | 2 | 0 | – | F | 2 ≠ 0 |
| 3 | `JPE Done` | 2 | 0 | – | F | 标志为 False，不跳 |
| 4 | `DEC ACC` | 1 | 0 | – | F | 计数减 1 |
| 5 | `STO 201` | 1 | 0 | 1 | F | 存计数 |
| 6 | `LDD 200` | 0 | 0 | 1 | F | 装入乘积 |
| 7 | `ADD 100` | 5 | 0 | 1 | F | + 5 |
| 8 | `STO 200` | 5 | 5 | 1 | F | 存乘积 |
| 9 | `LDD 201` | 1 | 5 | 1 | F | 恢复计数 |
| 10 | `JMP LOOP` | 1 | 5 | 1 | F | 循环回去 |
| 11 | `CMP #0` | 1 | 5 | 1 | F | 1 ≠ 0 |
| 12 | `JPE Done` | 1 | 5 | 1 | F | 不跳 |
| 13 | `DEC ACC` | 0 | 5 | 1 | F | 计数减 1 |
| 14 | `STO 201` | 0 | 5 | 0 | F | 存计数 |
| 15 | `LDD 200` | 5 | 5 | 0 | F | 装入乘积 |
| 16 | `ADD 100` | 10 | 5 | 0 | F | + 5 |
| 17 | `STO 200` | 10 | 10 | 0 | F | 存乘积 |
| 18 | `LDD 201` | 0 | 10 | 0 | F | 恢复计数 |
| 19 | `JMP LOOP` | 0 | 10 | 0 | F | 循环回去 |
| 20 | `CMP #0` | 0 | 10 | 0 | T | 0 = 0 |
| 21 | `JPE Done` | 0 | 10 | 0 | T | 标志为 True，跳 |
| 22 | `LDD 200` | 10 | 10 | 0 | T | 装入最终乘积 |
| 23 | `OUT` | 10 | 10 | 0 | T | 输出 10 |
| 24 | `END` | 10 | 10 | 0 | T | 停 |

最终状态：ACC = 10，M[200] = 10，输出 10。循环体跑了两次（对应乘数 2 的两个单位），每次加上被乘数 5，得到 5 × 2 = 10。

## 8. 位操作

### 二进制移位

移位把一个值的所有位向左或向右移动一位或多位。共有三种。

**逻辑移位（logical shift）** ——空出来的位补 0，被挤出的位丢弃。左移和右移对称。

```text
LSL #1   0001 0011  (19)   ->   0010 0110  (38)      ; 19 x 2
LSR #1   0010 0110  (38)   ->   0001 0011  (19)      ; 38 / 2
```

**算术移位（arithmetic shift）** ——用于有符号数。右移时把符号位（最左位）复制到空出来的位置，所以负数仍是负数。这是对一个有符号值减半的正确方式。

```text
ASR #1   1001 0110  (有符号负数)
      -> 1100 1011                        ; 符号位 1 被复制进来
```

**循环移位（cyclic / rotate shift）** ——不丢失任何位。从一端挤出的位被搬到另一端。

```text
循环右移 1   1011 0011   ->   1101 1001     ; 末位绕到最前
循环左移 1   1011 0011   ->   0110 0111     ; 首位绕到最后
```

### 位掩码：测试与置位

**位掩码（bit mask）** 是与按位运算符配合使用、用来触及某一位的位模式。程序监视或控制设备就靠它：输入或输出字节的每一位代表一个传感器或一个开关。

**用 AND 测试某位。** 用一个只在目标位为 1、其余位为 0 的掩码与该值做 AND；结果非零，说明那一位是 1。

```text
值     1011 0110   第 5 位是否为 1?
掩码 & 0010 0000   (第 5 位 = 1)
结果   0010 0000   非零 -> 第 5 位是 1
```

**用 OR 置位（置 1）。** 用一个在目标位为 1、其余位为 0 的掩码与该值做 OR；其余各位原样不变。

```text
值     1011 0010   置位第 3 位
掩码 | 0000 1000
结果   1011 1010   第 3 位变 1, 其余不变
```

实际中 CPU 走的是读-改-写：先读出当前端口值，应用掩码，再写回，这样置位第 3 位时不会干扰其他七个设备位。用掩码的反码做 AND 可以清零某位，用掩码做 XOR 可以翻转某位。

## 做题顺序

1. 遇到任何 CPU 题目，先按作用说出部件（ALU 算、CU 指挥、时钟同步、IAS 在 CPU 内存储），再补充细节。
2. 遇到寄存器题，先把唯一的通用工作寄存器 ACC、变址寄存器 IX 和其他专用寄存器分开，绝不要把 MAR（地址）和 MDR（数据）搞混。
3. 描述取指-执行周期时，用寄存器传送记号写出来，并按 PC → MAR → MDR → CIR → PC+1 的顺序逐项打勾。
4. 追踪汇编时，给每个寄存器和相关内存单元各开一列，每执行一条指令填一行。
5. 追踪前先看清循环测试的条件，并明确 JPE/JPN 实际读的是哪个标志。
6. 做位操作时，把掩码和值都以完整二进制并排写出，让 AND/OR 一目了然，不要在脑子里算。

## 常见错误

- 把 MAR 和 MDR 搞混。MAR 永远放地址，MDR 永远放数据。
- 取指时忘了把 PC 加 1，导致下一次又取同一条指令，永远循环。
- 把 `JPE` 理解成"为正则跳"。它的意思是**比较标志为 True 时跳**，即上一次 `CMP` 比出两个值相等。
- 把寄存器或缓存笼统叫"内存"而不区分主存。IAS 在 CPU 内部，RAM 在外部。
- 搞混寻址方式——尤其是间接寻址（地址里存的是地址）与变址寻址（地址加上 IX）。
- 只做一遍汇编，于是无法解决向前跳转。两遍正是为了处理向前引用。
- 对有符号数用逻辑右移却声称减半；要用算术右移才能保留符号位。
- 用 AND 去置位。AND 只能清零或保持，置 1 要用 OR。
