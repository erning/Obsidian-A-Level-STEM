# 20 Further Programming

## Core Idea

This A Level topic extends section 11 with programming paradigms (low-level, imperative, object-oriented, declarative) and the file processing and exception handling skills assessed in the Paper 4 practical.

## Source Alignment

- 9618 A Level section 20 Further Programming
- 20.1 Programming Paradigms
- 20.2 File Processing and Exception Handling
- Paper 3 and Paper 4 (except low-level and declarative programming)

## What to Learn

- What a programming paradigm is; the characteristics of low-level, imperative (procedural), object-oriented, and declarative paradigms.
- Low-level programming: writing code that uses addressing modes (immediate, direct, indirect, indexed, relative).
- Imperative (procedural) programming: variables, constructs, procedures, and functions (assumed from section 11.3).
- Object-oriented programming: terminology (objects, properties/attributes, methods, classes, inheritance, polymorphism, containment/aggregation, encapsulation, getters, setters, instances); designing classes; writing OOP code.
- Declarative programming: writing facts and rules from supplied information and satisfying a goal using them.
- File processing: opening (read, write, append) and closing a file; reading and writing records; operations on serial, sequential, and random files.
- Exceptions: what they are, why handling matters, when it is appropriate, and writing code that handles them.

## How to Study

- Place this after [[30 Computer Science/01 Topics/11 Programming/00 Overview|11 Programming]] and [[30 Computer Science/01 Topics/13 Data Representation/00 Overview|13 Data Representation]].
- For OOP, model a small system with classes before writing any code.
- Practise Paper 4 in one chosen language (Java, Visual Basic .NET, or Python) so that syntax becomes automatic.

## Practice Directions

- Write low-level code using each addressing mode.
- Design classes and write OOP code that demonstrates inheritance and encapsulation.
- Write a declarative solution using facts and rules.
- Write code that opens, reads, writes, and closes files of each organisation.
- Write code that raises and handles an exception.

## Common Traps

- Mixing paradigms inside one solution.
- Forgetting to close a file after processing.
- Catching an exception too broadly and hiding the real fault.
