# Counting and Probability Review Checklist

Use this after the worked examples and practice problems. Revisit [Counting and Probability Lecture Notes](10%20Lecture%20Notes.md) if any item is uncertain.

## Ideas to Recall

- [ ] A permutation is ordered; a combination is unordered.
- [ ] Repeated identical objects require division by repeated factorials.
- [ ] Restrictions in a line often become block or complement problems.
- [ ] Probability calculations start from a defined sample space.
- [ ] Complements are useful for "at least one" events.
- [ ] Mutually exclusive events cannot happen together.
- [ ] Independent events satisfy $P(A\cap B)=P(A)P(B)$.
- [ ] Conditional probability changes the sample space.
- [ ] Tree diagrams multiply along branches and add across alternative paths.

## Skills to Perform

- [ ] Decide whether order matters before choosing ${}^nP_r$ or $\binom nr$.
- [ ] Count arrangements with repeated letters.
- [ ] Handle together and not-together restrictions in a line.
- [ ] Enumerate dice or small equally likely sample spaces.
- [ ] Use combinations to calculate selection probabilities.
- [ ] Build a tree diagram for sampling with or without replacement.
- [ ] Use $P(A\mid B)=\frac{P(A\cap B)}{P(B)}$ in simple cases.
- [ ] Test independence using $P(A\cap B)$ and $P(A)P(B)$.

## Common Errors to Avoid

- [ ] Using permutations when combinations are needed.
- [ ] Counting arrangements of repeated objects as if all objects were distinct.
- [ ] Adding overlapping cases without subtracting the overlap.
- [ ] Treating without replacement as if probabilities stay constant.
- [ ] Confusing mutually exclusive events with independent events.
- [ ] Reversing $P(A\mid B)$ and $P(B\mid A)$.
- [ ] Using a shortcut before checking that outcomes are equally likely.

## Ready to Move On When

- [ ] I can describe the experiment and event before calculating.
- [ ] I can count outcomes without omissions or double counting.
- [ ] I can choose between listing, a table, a tree diagram, a Venn diagram, permutations, and combinations.
- [ ] I can solve conditional probability questions from a tree, table, or Venn diagram.
- [ ] I can explain whether two events are exclusive, independent, both, or neither.
